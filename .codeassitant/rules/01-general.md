# Правила создания текстов в блоге AlexJameson.github.io
Здесь приведены общие правила для статей на технические темы. Правила направлены на создание понятных, легко читаемых и фактически достоверных статей. Основной язык — русский. Используется GitHub Flavored Markdown и генератор Zola.

## Общие правила
* В начале файла должны быть метаданные в формате TOML. Обязательные поля — title, date, description, tags.

## Ссылки
* Ссылки из одного документа на другой внутри документации должны быть относительными по правилам Zola. Для статей путь выглядит так: @/articles/http-api-quickstart.md.
* На внешние ресурсы ставятся абсолютные ссылки.
